1. Введение

Lumen — Cinematic Prompt Studio — это локальное web-приложение для создания качественных промтов к моделям генерации видео (Seedance 2.0, Runway, Kling, Luma и др.). Приложение работает целиком в браузере, без сервера, и хранит все данные локально.

Что умеет

  • Собирать сложные кинематографичные промты по структурированным полям (subject, action, scene, camera, lighting, mood и др.)
  • Генерировать промт по короткой идее одним кликом через AI
  • Распознавать содержимое референсных картинок (Image-to-Video) с помощью AI Vision
  • Переводить с любого языка на кинематографичный английский
  • Создавать многосценные сценарии (Story Mode) с консистентным героем
  • Проверять сценарии на разрывы (Continuity Checker)
  • Экспортировать таймлайн в DaVinci Resolve / Final Cut Pro / Premiere через FCPXML
  • Сохранять историю, избранное, банк собственных героев и сцен

Что нужно

  • Современный браузер: Chrome, Edge, Firefox, Safari (последние 2 версии)
  • Файлы приложения: index.html и app.js в одной папке
  • Опционально: API-ключ OpenAI (или совместимого провайдера) для AI-функций
💡 Совет AI-функции опциональны. Без ключа приложение работает как ручной конструктор промтов — все базовые возможности доступны.

2. Запуск приложения

Локально

  1. Положи index.html и app.js в одну папку (например C:\Users\имя\Desktop\PROMT\).
  2. Дважды кликни на index.html — откроется в браузере по умолчанию.
  3. (Альтернативно) Открой через File → Open в браузере или перетащи в окно.
⚠ Важно Не переименовывай app.js — index.html ссылается на него по имени. Они должны лежать в одной папке.

На GitHub Pages / любом веб-сервере

Файлы статические, никаких сборок не требуется. Заливай оба файла в репозиторий → включай Pages → готова публичная ссылка.

3. Быстрый старт за 3 минуты

  1. Открой index.html в браузере.
  2. В поле «🎭 Subject (главный объект)» впиши: young woman with red hair
  3. В поле «🎬 Action»: walking through neon-lit city at night
  4. В поле «🌆 Scene»: cyberpunk Tokyo street, rainy reflections
  5. В выпадающем «🎨 Style» выбери cyberpunk.
  6. В выпадающем «💡 Lighting» выбери neon backlighting.
  7. Внизу справа в блоке «📜 Результат» нажми «📋 Скопировать (EN)».
  8. Готово — английский промт в буфере, вставляй в Seedance/Runway/Kling.
⚡ Ещё быстрее (1 клик с AI) Если настроил AI-ключ (раздел 5):
  1. В поле сверху «🎯 Цель» впиши на любом языке: реклама часов в стиле блокбастера
  2. Нажми 🎯 Goal mode.
  3. AI заполнит все поля английскими значениями. Жми 📋 Скопировать (EN).

3.5 🟢 Simple Mode (для блогеров и SMM)

Если ты не киномонтажёр и хочешь просто получить промт по короткой идее за 30 секунд — переключись в Simple Mode кнопкой в правом верхнем углу (🟢 Simple ↔ 🎛 Pro).

💡 Кому подходит Simple Mode SMM-специалисты, контент-менеджеры, блогеры, маркетологи, копирайтеры — все, кому нужен готовый промт для Sora/Runway/Kling/DALL·E, а не тонкая настройка 30 параметров. Опытные пользователи всегда могут перейти в Pro mode для дошлифовки.

3.5.1 📝 Text Mode (текст → профессиональный промт)

Самая мощная вкладка. AI извлекает «ядро идеи» (subject + action + object), фиксирует его, и строит вокруг него 10-блочный кинематографичный промт.

Шаги

  1. Открой Simple Mode → вкладка 📝 Текст.
  2. Введи идею на любом языке: мальчик смотрит мультик
  3. Выбери цель: 🎬 Видео или 🖼 Картинка.
  4. Выбери aspect ratio: 16:9, 9:16, 1:1, 4:3 и т.д.
  5. Выбери стиль (всего 12 стилей в сетке 4×3): 🎬 Кино · � Фото · 🎌 Аниме · 🧙 Фэнтези · 🤖 Киберпанк · 🎩 Нуар · 🎨 Иллюстрация · 📰 Документал · 🧸 Pixar · 🌸 Ghibli · � Hyperreal (UE5) · 🏛 Wes Anderson
  6. Выбери длину: Compact (50w), Standard (100w), Detailed (180w).
  7. Жми ✨ Создать.

Что появится на выходе

  • 🔒 Зафиксированное ядро — фиолетовая плашка показывает извлечённый subject + action + object. Это гарантия, что AI не уйдёт в сторону от твоей идеи.
  • 3 варианта с разной подачей: action-led / emotion-led / visual-led.
  • На каждом — score 0-100 (потенциал вовлечённости) и короткое обоснование.
  • Если AI потерял ядро — появится красное предупреждение и кнопка 🔒 Восстановить ядро.
📐 10-блочная структура промта Каждый промт строится по проверенной киношной схеме: subject · action · environment · camera · lens · lighting · palette · mood · style · technical. Это формат, который любят все современные видео-модели.

3.5.2 🖼 Image Mode (текст → картинка через DALL·E 3)

Вкладка для быстрой генерации картинок без сложных настроек.

Шаги

  1. Вкладка 🖼 Картинка.
  2. Опиши картинку на любом языке: уютная кофейня в Токио в дождь
  3. Выбери размер: 1024×1024 (1:1), 1792×1024 (16:9), 1024×1792 (9:16).
  4. Выбери стиль: Cinematic, Photorealistic, Anime, 🧸 Pixar, Watercolor, Cyberpunk, Minimal, Vintage и др.
  5. Жми 🖼 Сгенерировать.
🌐 Авто-перевод RU → EN Если ты ввёл идею по-русски, приложение сначала прогонит её через AI-переводчик в концентрированный английский (DALL·E работает с английским гораздо лучше), и только потом отправит в модель. На выходе ты увидишь оба варианта — что было отправлено и твой исходник.

Стили (10 шт)

🎬 Cinematic · 📷 Photorealistic · 🎨 Anime · 🧸 Pixar · 🖌 Watercolor · 🌃 Cyberpunk · ✏ Minimal · 📼 Vintage · 🎭 Oil Painting · 🌸 Pastel

Стиль добавляет специально оптимизированный suffix к промту. Pixar, например: "3D animation, expressive cartoon character, oversized eyes, soft subsurface scattering, octane render".

3.5.3 🎬 Video Mode (3 варианта по шаблону)

Вкладка для генерации 3 разных видео-промтов по выбранному шаблону формата.

Шаги

  1. Вкладка 🎬 Видео.
  2. Выбери шаблон формата (плитки): 📱 TikTok (9:16, 8с, хук), 📷 Reels (9:16, lifestyle), ▶ Shorts (9:16, retention), 🛍 Реклама (9:16, 6с, product-focus), 🎬 Кино (16:9, cinematic). Каждый шаблон уже содержит правильный aspect, длительность, стиль съёмки. Поля Brand и Message появляются только для шаблона «Реклама».
  3. Опиши идею: премиум-кофе утром на даче
  4. Опционально: Brand (название продукта/бренда) и Message (эмоциональный посыл).
  5. Жми ✨ Создать.

На выходе — 3 принципиально разных промта:

  • v1: action-led — упор на динамику, движение
  • v2: emotion-led — упор на эмоции, лица, чувства
  • v3: product/visual-led — упор на продукт и визуальную эстетику

Все 3 со score виральности 0-100 (отсортированы) и обоснованием на русском.

3.5.4 🌐 EN + RU — двуязычный вывод

С версии 10.4 каждый сгенерированный промт показывается на двух языках одновременно:

ЯзыкЗачемГде видно
🇬🇧 ENДля вставки в AI-модели (Sora, Runway, Kling, Veo, DALL·E). Английский — родной язык этих моделей, точность результата выше на 30-50%.Развёрнут по умолчанию, в выделенной рамке.
🇷🇺 RUДля понимания, что AI заложил в промт. Помогает увидеть, не потерялось ли ядро идеи. Удобно для отчётов и презентаций клиенту.Свёрнут под кнопкой ▼ RU — перевод для понимания.
🔄 Перевод, не пересказ RU-версия — точный перевод того же промта (тот же subject, action, те же блоки camera/lens/lighting), а не отдельный промт «по мотивам». Технические термины (anamorphic, dolly in, 24fps) сохраняются латиницей или в скобках.

Кнопки копирования

  • 📋 EN — копирует английский (используй для AI-моделей)
  • 📋 RU — копирует русский (используй для отчётов)

Все интеграции (🖼 → В картинку, 🖼 Превью, 🎬 → В Pro mode) автоматически передают английскую версию, потому что DALL·E и Pro mode тоже лучше работают с английским.

3.5.5 📦 Batch-экспорт всех вариантов

Под результатами генерации в Text Mode и Video Mode появляется панель быстрого экспорта в 4 форматах. Один клик — и все варианты в одном файле.

ФорматКому подходитЧто внутри
📋 MarkdownПрезентация клиенту, отчёт, README, Notion, ObsidianЗаголовки, метаданные, ядро идеи, все варианты с EN+RU блоками, score, обоснование
📊 CSVExcel, Google Sheets, Airtable, Notion-таблицыКолонки: #, title, score, approach, prompt_en, prompt_ru, why. С BOM для корректной кириллицы в Excel
💾 JSONРазработчики, бэкап, импорт в свои системыСтруктурированные данные с метаданными приложения, временной меткой ISO, всеми полями
📝 TXTBulk-paste в Sora/Runway, Telegram, SlackТолько английские промты столбиком с разделителями ---
📁 Имена файлов seedance-{text|video}-{ГГГГ-ММ-ДД_ЧЧММ}.{md|csv|json|txt} — никогда не перезатрутся, легко сортируются по дате.

Реальные сценарии

  • SMM-клиент — сгенерил 3 концепта рекламы → 📋 Markdown → отправил в Telegram готовым отчётом
  • Контент-менеджер — командный архив промтов → 📊 CSV → импорт в Notion базу данных
  • Разработчик — нужно встроить промты в свой пайплайн → 💾 JSON
  • Bulk-генерация — нужно прогнать 3 варианта через Sora подряд → 📝 TXT и копи-пейст по одному

3.5.6 🕘 История промтов

Все успешные генерации Text Mode и Video Mode автоматически сохраняются в локальную историю. Кнопка 🕘 История рядом с ✨ Создать открывает модальное окно.

Что хранится

  • Последние 50 генераций (FIFO — новая вытесняет самую старую)
  • В каждой записи: дата/время, режим, идея, ядро (subject+action+object), стиль, aspect, все варианты с EN+RU и score
  • Всё только локально в localStorage браузера — никуда не уходит

Что можно делать в истории

ДействиеКак
🔎 ПоискПоле сверху ищет по идее, ядру, стилю, заголовкам и тексту самих промтов (EN+RU). Фильтрует мгновенно при вводе.
👁 ОткрытьКлик по карточке (или кнопка) — модалка закрывается, переключается на нужную вкладку, варианты восстанавливаются полностью с EN+RU и кнопками
📋 ENБыстрое копирование лучшего варианта (с топ-score) — без открытия записи
🗑 на записиУдалить одну запись из истории
🗑 Очистить всёСнести всю историю (с подтверждением)
⌨ Управление модалкой Esc или клик вне окна — закрыть. Скролл внутри модалки, заголовок и поиск зафиксированы.
💡 Зачем это нужно
  • Случайно закрыл вкладку — варианты не пропали, найди и восстанови
  • Клиент через 2 дня просит «тот промт про Pixar» — за 5 секунд найдёшь по поиску
  • Сравнение разных подходов: открой 2 записи и посмотри, как менялась идея

3.5.7 🔍 Img→Prompt (картинка → промт)

Самый мощный режим — кидаешь скриншот/фото/постер, AI разбирает его на части и выдаёт 3 готовых промта для генерации видео или картинки. Не нужно ничего вводить вручную, всё извлекается из эталона.

📌 Когда использовать
  • Увидел крутой кадр/референс — хочешь похожее видео или картинку
  • Клиент скинул mood-board — нужно превратить в техническое ТЗ
  • Нравится стиль художника/режиссёра — переноси на свой сюжет

Шаг 1. Загрузить картинку

Три способа:

  • Drag & drop — перетащи файл в drop-zone
  • 📂 Выбрать файл — кнопка открытия файла
  • 📋 Вставить из буфера — Ctrl+V прямо в зоне или Cmd+V на Mac (работает только когда вкладка I2P активна и фокус не в текстовом поле)

Поддержка: JPG, PNG, WebP. Рекомендуется до ~3 МБ — большие изображения сжимаются автоматически.

Шаг 2. Multi-Reference (опционально)

Можно добавить до 2 дополнительных картинок с конкретной ролью — тогда AI возьмёт от каждой только нужный аспект:

РольЧто возьмёт из reference-картинки
🎨 ПалитраТолько цветовую гамму, насыщенность, температуру
💡 СветТолько характер освещения, направление, контраст
📐 КомпозицияТолько расстановку планов, кадрирование, угол
🌙 НастроениеТолько эмоциональный тон, atmosphere
👤 СюжетТолько субъект, действие (без визуального стиля)
🎬 Полный стильВсё сразу — как вторая основа

Итого можно собрать до 3 картинок в одном запросе (1 главная + 2 ролевых). Например: главная = поза героя, ref1 = палитра Wes Anderson, ref2 = свет Roger Deakins.

Шаг 3. AI Vision Analysis

Кнопка 🔍 Анализ картинки отправляет изображение в GPT-4o Vision и возвращает 9 структурированных полей:

SubjectГлавный объект/герой
SceneЛокация, окружение
LightingТип, направление, температура света
MoodЭмоциональный тон
PaletteДоминантные цвета
CameraПлан, угол, объектив
Era / StyleЭпоха, художественное направление
CompositionРасстановка, правило третей, симметрия
NotesДополнительные нюансы

Каждое поле можно отредактировать вручную прямо в панели — это влияет на финальные промты.

Шаг 5. Target и Aspect

  • 🎬 Видео — промт для Sora / Runway / Kling / Seedance. Aspect: 9:16, 16:9, 1:1, 4:5, 21:9
  • 🖼 Картинка — промт для DALL·E / Midjourney. Aspect: 1:1 (1024×1024), 16:9 (1792×1024), 9:16 (1024×1792)

Шаг 6. Генерация и работа с результатом

Кнопка ✨ Сгенерировать выдаёт 3 варианта с разной интерпретацией. Каждый вариант:

  • 📋 Копировать — скопировать промт в буфер
  • ⭐ 1–5 — поставить рейтинг (сохраняется в истории)
  • ❤ В избранное — отметить как фаворит
  • → Pro Mode — отправить выбранный вариант в полную форму для доработки
⚠ Стоимость
Vision Analysis = ~1 запрос к GPT-4o (~$0.01 за картинку). Генерация 3 вариантов = ещё ~1 запрос. Итого ~$0.02 за полный цикл. Multi-reference немного дороже из-за большего payload.
💡 Workflow «увидел → получил видео»
  1. Скриншотишь крутой кадр из фильма
  2. Drag в I2P → 🔍 Анализ → правишь поля под свой сюжет (например меняешь Subject)
  3. Выбираешь 🎨 Стиль-реф + 🎬 Видео + 16:9
  4. ✨ Сгенерировать → ставишь ⭐ лучшему → → Pro Mode
  5. В Pro Mode добавляешь multi-shot и timeline → копируешь финальный промт в Sora

3.5.8 🔒 Face Lock (сохранение лица 1:1) v15.5

Типичная боль при работе с AI-видео: загружаешь фото человека → модель интерпретирует лицо, немного «додумывая» черты. На выходе герой похож, но не тот. Face Lock впрыскивает в промт строгую инструкцию сохранить лицо 1:1 — идентичные черты, кожа, глаза, причёску, челюсть.

Где включить

Два переключателя, синхронизированных между собой через localStorage.lumen_face_lock:

ГдеРасположение
Pro Mode → 🖼 Image-to-VideoФиолетовый callout под полем Motion prompt
Simple Mode → 🔍 Img→PromptФиолетовый callout между AI-анализом и выбором режима

Включил в одном месте — автоматически включён и во втором. Состояние сохраняется между сессиями.

Что попадает в промт

В Pro Mode I2V появляется фраза:

IMPORTANT FACE-LOCK: preserve the EXACT same face from the reference image
with 100% fidelity — identical facial features, skin tone, eye color and shape,
nose and lip shape, hairstyle and hair color, jawline, and overall likeness.
This is the SAME person, not similar. No face morphing, no identity drift,
no stylization of facial features. Reference-accurate portrait, 1:1 character
consistency throughout.

В I2P в system-prompt модели добавляется директива FACE-LOCK MODE ENABLED (critical override) — AI обязан включить в каждый из 3 сгенерированных вариантов явные фразы вроде "exact same face as reference", "1:1 likeness preservation", "no face morphing". Это работает даже в режимах «Стиль-реф» и «Кинематично», где обычно лицо стилизуется.

Совместимость с AI-видео моделями

МодельЭффект Face Lock
Runway Gen-3 / Kling / Seedance 2.0Сильный — у них есть native character-consistency, фраза усиливает
Sora / Luma Dream MachineСредний — интерпретативные модели, но фраза помогает удержать identity
DALL·E 3 / MidjourneyСредний — для статических картинок работает хорошо в режиме Img→Image
⚠ Это prompt-engineering, не технический lock
Face Lock — это усиленная текстовая инструкция модели. Настоящий 100% lock даёт только InstantID / IP-Adapter на уровне модели, чего большинство API не предоставляют. С Face Lock результат стабильно лучше, но не гарантирован 100%.

3.5.9 🎬 Motion Hints (подсказки движения) v15.6

Для Image-to-Video качество анимации сильно зависит от описания движения. Без него модель додумывает — часто в сторону teleporty/morphy артефактов. Motion Hints даёт 6 проверенных пресет-фраз и AI-генератор motion-описания на основе референс-кадров.

Где находится

Pro Mode → 🖼 Image-to-Video — над полем Motion prompt появилась строка с кнопками.

Пресет-чипы (мгновенная вставка, без AI)

ЧипДля какого типа кадраЧто вставляет
🫧 BreathПортреты, говорящие головыЕстественное дыхание, моргание, микро-мимика
📷 Push-inДраматические крупные планыМедленный наезд камеры, dolly forward, углубление фокуса
💨 WindНа улице, природа, тканиЛёгкое покачивание волос и ткани, дрейф частиц в воздухе
🌫 ParticlesИнтерьеры со светомПылинки в лучах света, волюметрическая дымка
🌊 ParallaxПейзажи, wide shotsПередний план движется быстрее заднего — эффект 3D глубины
🎥 DollyTracking shotsБоковое смещение камеры, субъект в центре

Клик по чипу — фраза добавляется к текущему motion (с запятой), перегенерируется английский вывод.

✨ AI подсказать

Отдельная кнопка рядом с чипами. Работает в двух режимах:

  1. Только первый кадр — AI Vision анализирует тип сцены и сам выбирает подходящее движение: портрет → breath+blink, пейзаж → wind+particles+light shift, интерьер → dust+camera push, action → parallax/tracking.
  2. Первый + последний кадр (interpolation) — AI анализирует различия между двумя ключевыми кадрами (поза, выражение, позиция, свет, угол камеры) и пишет переходный narrative вроде: «hair shifts from left to right as subject slowly turns head, wind intensifies, golden hour deepens into dusk».
💡 Workflow на практике
  1. Загрузил первый кадр → runVision автоматически заполняет subject/scene/lighting
  2. Клик ✨ AI подсказать для motion → получил motion-описание под тип сцены
  3. Если есть последний кадр — добавь его, AI напишет интерполяционный narrative
  4. Дополни чипами (напр. Wind если работаешь с природой + Particles для атмосферы)
  5. 🔒 Face Lock включай если в кадре есть человек с узнаваемым лицом
  6. Копируй EN → Sora/Runway/Kling/Seedance

5.2 CORS proxy (для Nekocode и других gateway'ев)

Если твой провайдер (Nekocode, Azure, кастомный gateway) не разрешает CORS — подними свой прокси на Cloudflare Worker. Бесплатно, 100k запросов/день, ~5 минут setup.

  1. Зайди на dash.cloudflare.com → Workers & Pages → Create application → Create Worker
  2. Имя: lumen-proxy (любое) → Deploy
  3. После деплоя жми Edit code → удали весь дефолтный код (Ctrl+A → Delete)
  4. В Lumen открой ⚙ AI → нажми жёлтую кнопку 📋 Шаблон Cloudflare Worker — код скопируется в буфер
  5. Вернись в редактор Cloudflare → вставь (Ctrl+V) → жми Deploy
  6. Скопируй URL воркера (вида https://lumen-proxy.твой-ник.workers.dev)
  7. В Lumen ⚙ AI:
    • CORS proxy → вставь URL воркера
    • Жми Nekocode (или другого провайдера) — Base URL / Model / Auth автозаполнятся
    • API Key → ключ провайдера
    • Сохранить

Как это работает: Браузер → твой Worker → Nekocode → обратно через Worker (он добавляет Access-Control-Allow-Origin: *). Ключ проходит только через твой Worker — не через публичный прокси.

Проверка прокси: открой в браузере https://твой-воркер.workers.dev/https://api.github.com — должен прийти JSON от GitHub.

💡 Альтернативы прокси: Vercel/Netlify Function (serverless, бесплатно), свой node-сервер с cors middleware, локальный запуск Lumen в Electron/Tauri (CORS не применяется).

5.3 Какую модель выбрать

ЗадачаРекомендацияПочему

5.4 Безопасность ключа

Ключ не передаётся никуда кроме указанного API. Хранение:

  • Ключ обфусцируется через XOR + base64 и сохраняется в localStorage с префиксом XOR:
  • На каждом AI-запросе расшифровывается на лету
  • Никогда не появляется в логах в plaintext
⚠ Это обфускация, не криптография Защита от случайного просмотра localStorage. Если кто-то может выполнить JS на твоём компьютере — ключ всё равно вытащат. Не используй приложение на чужой машине.

7. Image-to-Video режим

Если у тебя есть референсная картинка и ты хочешь чтобы видео началось с неё — переключись в режим I2V.

Шаги

  1. Внизу формы переключи режим: T2V → I2V.
  2. Перетащи картинку в зону «🖼 Первый кадр» (или нажми «выбрать файл»).
  3. Если AI-ключ настроен — Vision автоматически распознает картинку и заполнит поля subject/scene/style/lighting и др.

10. Story Mode (multi-scene)

Самая мощная фича — превращает одну идею в N связанных сцен с консистентным героем.

Открыть

Кнопка 🎬 Story в шапке справа, или Command Palette → Story Mode.

Шаги

  1. В поле «💡 Большая идея» впиши концепт на любом языке: детектив расследует исчезновения в туманном порту
  2. Выбери жанр/тон: Neo-noir, Cyberpunk thriller, Anime adventure и др. (11 опций)
  3. Слайдером «🎚 Кол-во сцен» выбери от 3 до 10

  3. Выбери story.fcpxml
  4. В диалоге «Load XML»: укажи Source clips path = папка с распакованным ZIP
  5. Галочки оставь по умолчанию → OK

После импорта

  • В Media Pool появятся PNG-превью + объект таймлайна с именем твоей истории
  • Двойной клик по таймлайну → открывается на дорожке
  • Видно N клипов подряд, подписанных, с маркерами continuity

Замена placeholder на реальное видео

  1. Сгенерь mp4 через Seedance/Runway/Kling (используя промт из <note>)
  2. Перетащи mp4 в Media Pool
  3. Зажми Alt (Win) / Option (Mac) и перетащи новый клип на placeholder — Resolve заменит сохранив длину/позицию
  4. Альтернатива: правый клик на клипе → Replace Clip

Final Cut Pro

Premiere Pro

19. Экспорт / импорт состояния

Экспорт

  • 📋 EN / 📋 RU — английский или русский промт в буфер (кнопки рядом с результатом)
  • .txt — выгружает только текст промта в .txt файл
  • .md — Markdown с метаданными, структурой и всеми вариантами
  • .json — полное состояние формы и результата (кнопка { })
  • 🔗 Share (иконка в шапке) — кодирует всё состояние в URL после #s=... и копирует в буфер
  • � Compare (иконка в шапке) — сравнение двух промтов side-by-side с подсветкой различий

Импорт

  • Open URL с #s=... — приложение автоматически распакует и применит состояние при открытии
  • Автозагрузка из localStorage — последнее состояние формы восстанавливается при каждом открытии (без явной кнопки)
  • Импорт из ранее выгруженного .json-файла через UI пока не реализован — используй URL-Share для переноса между устройствами.

22. Решение проблем

AI-кнопки нажимаются, но ничего не происходит
1) Проверь что в ⚙ AI вписан ключ. 2) Открой DevTools (F12) → Console → ищи красные ошибки про 401/403/CORS. 3) Проверь что хватает $ на балансе у провайдера.
Vision не распознаёт картинку в I2V
Vision требует gpt-4o или gpt-4-vision-preview. Зайди в ⚙ AI → смени модель. gpt-4o-mini работает плохо для Vision.
Story Mode возвращает «AI вернул не сценарий»
Локальные модели (Ollama) часто не держат JSON-формат. Используй gpt-4o или claude-sonnet. Слабые модели (mini) тоже могут портить JSON — переключись на старшие.
localStorage заполнился, ошибка quota
Превью-картинки в Story Mode тяжёлые. Очисти историю/избранное (🗑) или удали старые сценарии. Размер квоты браузера — обычно 5–10 MB.
FCPXML импортируется в DaVinci, но клипы красные («Media offline»)
Resolve не нашёл папку media/. Правый клик в Media Pool → Relink Selected Clips → укажи папку с распакованным ZIP.
Premiere ругается на FCPXML 1.10
Premiere поддерживает только до 1.8. Импортируй в Resolve → File → Export → Final Cut Pro 7 XML → этот файл откроется в Premiere.
Голосовой ввод не работает
В Firefox/Safari Web Speech API ограничен. Используй Chrome или Edge. Также нужен HTTPS или localhost — на file:// может не работать.
Поля заполняются «[object Object]»
Старая версия. Убедись что у тебя app.js v7+. Это было исправлено через функцию applyAiFields.
Кнопка ★ Избранное не работает
Сначала нужно сгенерировать промт (нажать любую кнопку чтобы заполнился блок результата). Без английского промта избранное пустое.
Хочу сбросить всё к заводским настройкам
F12 → Console → введи localStorage.clear() → Enter → перезагрузи страницу. Сотрутся все настройки, история, ключ, темы, story.

23. FAQ

Сколько стоит использовать?
Само приложение бесплатно. AI-функции — по тарифам твоего провайдера. С gpt-4o-mini один промт ≈ $0.0001, story из 5 сцен ≈ $0.001. Превью DALL·E ≈ $0.04 за кадр. Локальный Ollama — $0.
Можно ли работать без интернета?
Да. Без AI — полностью офлайн. С Ollama (локальная модель) — AI тоже офлайн.
Где хранятся мои данные?
Только в localStorage твоего браузера. Никаких облаков, никакой телеметрии. Очистка кэша браузера = потеря данных.
Можно поделиться сценарием с коллегой?
В Story Mode → 📦 Экспорт ZIP или 📄 Экспорт .txt. Коллеге пришли файл — он откроет.
Поддерживается ли мобильный?
Интерфейс адаптивный, но лучше работать с десктопа. Голосовой ввод и drag-drop для I2V на мобильных ограничены.
Можно сделать публичную ссылку?
Да — залить два файла на GitHub Pages, Netlify, Vercel. Папка статическая, сборки не нужно.
Где брать API-ключи?
OpenAI: platform.openai.com/api-keys. Anthropic: console.anthropic.com. OpenRouter: openrouter.ai/keys. Ollama: установи с ollama.com, ключ не нужен.
Какая модель Seedance/Runway лучше?
Промт-структура одинаковая для всех. Skip-test: возьми один и тот же промт из приложения и прогони через несколько генераторов — выбирай по результату.

Контакты и обновления

Это локальное приложение. Обновления — заменой файлов index.html, app.js, extras.js и папки js/ (модули фич: i2p.js, story.js, history.js, continuity.js, fcpxml.js).

📂 Структура проекта (v15.7)
  • index.html — DOM, стили, точка входа
  • app.js — ядро (Pro Mode, AI-слой, Simple Mode core, Text→Pro, Presets, Text-to-Image)
  • extras.js — skeleton-loaders, templates gallery, onboarding tour, PWA
  • js/i2p.js — Image-to-Prompt pipeline (vision, multi-ref, генерация)
  • js/story.js — Story Mode (multi-scene generator)
  • js/continuity.js — AI script supervisor для Story Mode
  • js/fcpxml.js — FCPXML экспорт таймлайна
  • js/history.js — Prompt History (общая история Simple Mode + I2P)
Все модули загружаются через <script defer> в порядке зависимостей. Глобальные функции ($, aiCall, toast) определены в app.js и доступны всем модулям.
